Working on feature branch- (first time)
1. First fork the repo
2. Clone the forked repo into local
3. cd forked-repo
4. git remote add upstream <lnk to actual repo>
 (Note origin=fork, upstream=original repo(
5. git fetch upstream
6. git checkout main
7. git rebase upstream/main
8. git checkout -b <feature-branch-name>
9. git add . && git commit -m "Added new files"
10. git push origin <feature-branch-name>
11. Go to GitHub ui and do pull request to do changes from feature branch to main

Next time when working on same branch: 
1. cd to repo
2. git checkout <feature-branch-name>
3. git pull origin <feature-branch-name>
4. git checkout main
5. git fetch upstream && git rebase upstream/main
6. git add .
7. git commit -m "Message"
8. git push origin <feature-branch-name> --force (Use --force if needed)

Always note before doing push..if some time passes please do: git fetch upstream && git rebase upstream/main so that no conflicts happen
